Panel Name

fwAccessControl/fwAccessControl_EditDomain.pnl
Introduction

This panel is used to display and edit the details concerning the domain.

Figure 1: Edit Domain Panel in edit mode.

The three text fields define domain short name, domain full name and domain comment. It is the domain short name that identfies the domain.
The non-editable text-field "id:" in the top-right corner presents the numerical, unique domain index, as used internally by PVSS (it is PVSS area identifier). The local system domain has this identifier equal to -1.
The list presents the names of privileges in the domain, with corresponding numerical privilege levels.

The same panel is also used when a new domain is created; in this case the Holders... button is not available, and Apply button is replaced by Create button:

Figure 2: Edit Domain Panel in create mode.

If current user has no privileges to edit domain details, the panel is open in "view mode", where editing operations are disabled:

Figure 3: Edit Domain Panel in view mode.
Instructions
To show the holders of certain privilege in the domain:
Select a privilege in the list, then click on Holders... button; this will bring the Privilege Holders List panel. Note that this button is not available in "create" mode.
To reset the changes made in this panel:
Click on Reset... button; The fields in the panel will reflect again the current settings for the domain. In the "create" mode it will clear all fields and set default privilege names.
To create a new domain:
In the panel running in "create" mode, input at least a valid name for a new domain, then press Create button.
To change the full name or comment of the domain:
Edit the data in corresponding text fields, then press Apply button.
To rename the domain:
Edit the data in the "Name" text field, then press Apply button.
To edit privilege names in the domain:
Edit the names directly in the list of names, then press Apply button.
Restrictions
  1. It is not possible to modify the name, full name and comment of the system domain
  2. The names for privileges 29,30 in the system domain are hard-coded and cannot be modified
  3. When working with UNICOS access control scheme the following apply:
    1. Domain full name and comment cannot be changed
    2. There are only 4 privilege levels
    3. A domain cannot be created or deleted
