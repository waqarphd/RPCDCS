Panel Name

fwAccessControl_EditGroupPrivileges.pnl
Introduction

This panel is used to configure the privileges granted to a group.

The privileges which will be granted to the group are presented in the list on the right.

Figure 1: Edit Group Privilege panel when opened - the domain is not yet selected in the left list.

The list on the left will be filled with privilege names, when a domain is selected in the "Domain" combo-box above. Initially, when the panel is open - see Figure 1 - no domain is yet selected ("Select Domain") text appears in the combo-box and the list of privileges is empty. When a domain is selected, the left list is populated with the names of privileges in this domain - see Figure 2.

Figure 2: Edit Group Privilege panel: multiple selections in both list are possible.

The buttons between the two lists are used to grant and revoke privileges (see Instructions for more details).

On both lists, operations using multiple-selection is possible - selected items are "highlighted". When a certain privilege appears already in the list on the right side, it is not possible to select it on the left side - these items have grey background on the list (in Figure 2 these are "Operate" and "Administer", because "Demo:Operate" and "Demo:Administer" are already in the list on the right side).

The list of all privileges granted to a group may be long, therefore a filter may be applied to the list on the right - when the Show all granted privileges box is unselected, only the privileges in the domain selected in the combo-box above the left list are shown, as in the example on Figure 3:

Figure 3: Edit Group Privilege panel: the list on the right displays only the privileges in the domain selected in the list on the left; the Show all granted privileges button is unselected.
Instructions
To select the domain:
Use the "Domain" combo box
To add privilege(s) in selected domain:
Select the domain, then select the privilege(s) in the left-side list (multiple selection is possible: hold CTRL key while clicking on the item), then click the   >   to add the privilege(s); to add all privileges in selected domain one may use the  >>  button. The privileges that were added will appear in the righ-side list and will be greyed-out in the left-side list.
To remove privilege(s):
Select the privileges in the right-side list (multiple selection is possible: hold CTRL key while clicking on the item), then click the   <   to remove the privilege(s). The privileges which were removed disappear in the righ-side list and will appear un-greyed in the left-side list.
To filter displaying of privileges:
Use the Show all granted privileges button. When it is on (default) all the privileges are displayed in the right-side list. When it is off, only the privileges in selected domain are displayed.
To reset the changes done in this panel:
Click on the Reset button. This will return the panel to the state it had when opened.
To apply the changes and close this panel:
Click on the OK button. Note that the changes will not be applied until you apply all changes to the group in Group Edit panel by clicking the OK button there.
To cancel the changes and close this panel:
Click on the Cancel button.
Restrictions
  1. System privilege 1 (Visualize) is required for each user to log in. This privilege is therefore enforced for each group.
  2. Due to the way the access control for areas is implemented in PVSS, each user needs to have the privileges in system domain to "unmask" the actual privileges in the areas. Therefore all users need to have privileges 20 - 28 in the system domain - these privileges are thus enforced by the tool for each group.
