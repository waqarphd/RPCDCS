Panel Name

fwRDBArchivingGeneral.pnl
Introduction

This panel is used to set up the database login information for RDB Archiving, and which mode the database is in.

If the database is in "Custom" mode then the user can use the "Configure Mappings..." button to open fwRDBArchivingMappings.pnl.
Instructions

Before running the RDB Archive Manager for the first time, or if you want to change the database details, you should enter your database name, username, and password into this panel and then click the "Set as Archive RDB" button. This will set these values in the three places where PVSS looks for them. You should restart the RDB Archive Manager after changing the database details. If you reopen the panel, the database name and username will be filled out automatically, but not the password. You only need to type the password again if you need to change it, or if you want to view or change the database mode or mappings.

To check which mode the database is currently in, enter the database details and click "Connect". Note that this only connects to the database for using this panel and fwRDBArchivingMappings.pnl, it does not start archiving. If the connection could be opened, the panel will show "Current Mode: Automatic", "Current Mode: Custom", or "Current Mode: Off" and enable the buttons to switch modes. If the current mode is Off, it may mean that some features from Automatic mode and some features of Custom mode have been enabled.

The writeWithBulk mode should either be "On" or "Off". These indicate that that the writeWithBulk setting in the configuration file agrees with the way the database is set up. It could also say "On in database" or "Off in database", which means that the setup in the database does not match the setting in the configuration file. In this case you should use the 'Turn on' button to turn writeWithBulk on in both places. Otherwise, data could be written twice or not at all. You should always make sure that the RDB Archive Manager is not running when you change the writeWithBulk setting.

To change the RDB Archiver settings such as buffer size, click the 'Settings...' button to open the standard PVSS setup panel for the RDB Archive Manager.

If the database is in Custom mode, the "Configure Mappings..." button will be enabled. You can click on it to configure which tables the data is stored in, using fwRDBArchivingMappings.pnl.

Opening the panel

This panel can either be opened directly, or by clicking the 'Parameterize...' button on the framework archiving setup panel, which sets $sDPE to the list of selected datapoint elements.

Datapoint Aliases

There is a table in the database called ALIASES, where changes to the aliases of archived datapoint elements are recorded. This is only automatically updated with changes to the actual elements which are archived. However, you may wish to record the aliases of the datapoints themselves, as these contain the names of devices in the FSM tree. These cannot be automatically recorded on the database side because PVSS only sends information about the specific elements being archived. If %sDPE is defined, then an 'Update Aliases Table' button will appear on this panel. If you click this then the aliases of the parent DPs of the DPEs you selected will be added to the database. If your database is in Automatic mode, you can also access this button from the fwRDBArchivingMappings.pnl

The button adds alias information for each datapoint when a mapping is added for one of its elements. If any of the aliases change, you can update the information in the database using the ‘Update Aliases table’ buttons. This updates the information for all datapoint elements which have mappings.

A new entry is only written to the ALIASES table when the alias has changed from the previous value in the table, or when the alias is not already in the table and is not blank.
Restrictions The "Set as Archive RDB" button is only enabled if you have changed the database information from what the archiver is currently using.

 

 

 

Dollar Parameters
Name Description  
$sDpe

The name of the data point element(s) to configure. While the settings on this panel are not datapoint-specific, this dollar parameter will be passed on to fwRDBArchivingMappings.pnl if the user clicks the "Configure Mappings..." button.

If you wish to use the panel for multiple data point elements, you should pass a list in the same format as the PVSS dyn_string, i.e.dpe1 | dpe2 | dpe3
