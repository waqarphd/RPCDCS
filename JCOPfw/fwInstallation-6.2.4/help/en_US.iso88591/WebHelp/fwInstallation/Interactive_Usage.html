JCOP FW Component Instalaltion Tool Documentation

Interactive Usage


Panel Name

fwInstallation.pnl
Introduction The installation tool's main panel allows for installation, modification, and deletion of JCOP Framework components as well as self-packaged components. These components are described in a xml description file, that can be created with the installation tool packager panel (fwInstallation/fwInstallation_Packager.pnl). For further information refer to the description on the Component Installation web pages.

At the first startup of this panel, you were asked to define a default installation directory, a project path of your project or a new one, which has subsequently been created. In order to change this, you have to go to the Advanced Options. The stored project path will be the destination of all installations with the tool.

Install Components

  • Look for new components in:
    Select the source directory for your installation (the last path will be saved and your default when clicking on the folder selector button).
  • Available Components:
    List of all components in the given source path with their version number. Clicking in the last column will change to "Install" for a previously not installed component or "Re-Install" for already installed components. "Installed" denominates already installed components.
  • Show also Sub-Components:
    Some components have sub-components, which are not installed stand-alone and mostly not usable stand-alone. When this box is checked, also sub-components are shown in the list and can be installed/re-installed. These components are prefixed with an underscore.
  • Refresh:
    Refreshed the list of available components.
  • Install:
    Perform the installation of the selected components. Before final execution, you will get a window showing the selected components, the destination directory and subsequently possible conflicts or missing required components.

View/Delete Components

  • Installed Components:
    List of all components in the given source path with their version number. Clicking in the second-last column will change to "Delete"  for already installed components. A click on the word "HELP" in the last column will display the help file defined by the author of the component.
  • Show also Sub-Components:
    Some components have sub-components, which are not installed stand-alone and mostly not usable stand-alone. When this box is checked, also sub-components are shown in the list and can be deleted. Deleting a sub-component can render any other component unusable! All sub-components are prefixed with an underscore.
  • Refresh:
    Refreshed the list of installed components.
  • Delete:
    Perform the deletion of the selected components. Before final execution, you will get a window showing the selected components.
