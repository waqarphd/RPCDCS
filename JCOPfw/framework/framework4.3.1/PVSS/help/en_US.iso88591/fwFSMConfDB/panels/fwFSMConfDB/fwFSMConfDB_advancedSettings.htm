Instructions
Loading settings:
These settings defines the behavior of the configurator when recipes are loaded.
- force configurator cache update
if active, the configurator will always overwrite the internal PVSS caches for a given running mode even if the mode was previously loaded and the caches already exist in PVSS.
- stop on error
if active, the caching of recipes is aborted if any of the devices handled by the configurator are NOT defined in the recipe.
- clear caches at initialization
if active, the caches created by the configurator after each load will be deleted at initialization of the FSM. As a result no recipes will be available before executing a new LOAD.
- Stop if no recipes found
if enabled, it sets the state of the configurator to ERROR if no recipes are found during the loading process for a given run mode.
