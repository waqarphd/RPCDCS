Panel Name

fwAlarmHandlingGroupsScreen.pnl
Introduction

This panel allows you to view the alarms in your system, and to have a summarized view of alarms, grouped following your criteria.
Instructions Compared to fwAlarmHandlingScreen.pnl, this panel features the same alarm list. The alarm filtering is still available, using the usual button "Modify Filter". Historical alarms filter is also available, inside the box "Mode".

The extra feature of this panel is mainly the Grouping: the alarms shown on the list are scanned and then a statistic is made:
  • How many alarms are visible
  • How many of them have not been acknowledged
  • How many of them are new
  • Time of the last alarm
The statistic is based on a criteria as specified by a user (i.e. take into account only alarms coming from "system1" and "system2", and having dp name matching the pattern "dp*"...). This criteria is saved under a Group. You can create as many groups as you wish. Each group will appear as a separate box.

Groups

To create a group, click on the button "Groups Setup". Please refer to fwAlarmHandlingGroupsConfiguration.

When a new alert displayed in the alarm list matches the criteria of a group, the group will increment the "New" counter. The "Unack" counter will be incremented only if the alarm is acknowledgeable and not yet acknowledged. The "New" counter will be incremented; it will be decremented only after the operator opened the alarm help (right-click on the alarm line, click on "Alarm Help").

The alarm matching is done in sequence: if it does not match the first group criteria, the second group criteria is then analysed, and so on. An alarm that matches a group will be ignored by all the following groups.

The colour of the group is inherited from the highest priority alarm. Among alarms of the same priority, the colour of the unacknowledged alarm is taken.

New alarms

Every time a new alarm appears, a bell is ringed (option - see fwAlarmHandlingGroupsConfiguration for more info).

An alarm appearing on the alarms list is marked as New if the operator has never opened its help (right-click on the alarm line, click on "Alarm Help").
