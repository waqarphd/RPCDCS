Panel Name

fwWienerDriverConfiguration.pnl
Introduction

This panel is used to create and delete devices to represent the Wiener OPC server used to connect to Wiener hardware.

It is also possible to create the config files for the OPC servers based on the devices which are existing in your PVSS system.
Instructions

When the panel loads, the configuration for the default Wiener Marathon driver number (14) is shown. By selecting a different number from the "Display configuration for driver number:" combo box, it is possible to view the configuration for a different Wiener driver number.

For each Wiener driver, it is possible to set a description and to create the XML file required for the OPC server.

To create an XML config file, first make sure that the correct Wiener driver number has been selected. Then, in the "OPC Config File" section of the panel, choose the file name (and path if required) in which to save the configuration. Next, go to the table where there is a list of all the relevant devices in the current system. For the KVASER OPC server, these devices will be Wiener CAN buses. For the Marathon OPC server, these devices will be Wiener Marathon power supplies.

For each device you can choose whether or not it should be included in the XML config file. This is done by click in the "Declare in Driver X file" column. The value will toggle between "TRUE" and "FALSE" each time you click in the cell. "TRUE" means that the device and all its children (channels etc.) will be included in the OPC server config file. When you have selected which devices to include, press the "Create Config File" button to create the config file.

In addition, in the top right of the panel there is the "Manage Drivers..." button which can be used to create, delete and configure the devices that represent the Wiener OPC servers.

NOTE: It is not required to have a Wiener Driver device in order to connect to the Wiener OPC server. It is only beneficial to help with the creation of the XML config file and allows for debugging by providing access to OPC items available at the server level (e.g. cache words, up-time etc.).
