Introduction

Access Control Server Console

This panel displays allows to monitor (and manage) client systems from the Access Control Server.
Warning: this panel is not yet completed. It will be improved in the next releases of the component
The panel is presented in Figure 1.

Figure 1: The Server Console panel

The panel consists of the following elements:

  • The indicator of the name of the machine on which this panel is running ("ACServer:" in Figure 1)
  • The indicator of the connection status (green "OK" in Figure 1)
  • The table containing the list of all systems in the distributed system. The first column indicates system number, the second: system name, the third shows the current setting of the AC Server on that system, and the last: indicates whether the system is currently being managed by the server.
Right-clicking on the table brings up a context menu. If clicked on the system machine corresponding to the server itself, the context menu will be such as the one in Figure 2:
Figure 2: Context menu for the server
Otherwise, for client machines, the menu presented in Figure 3 will pop up; the top line indicates that the line for which the menu was displayed corresponds to the AC Server.
Figure 3: Context menu for a client
Selecting the Set AC Server option in client's context menu, brings up the sub-menu presented in Figure 4. The top line indicates the name of the client system for which the context menu was displayed.
Figure 3: "Set AC Server" sub-menu of the context menu
Instructions
To refresh the display:
Press the Refresh button
To make a client be managed by this server:
Right-click on the line corresponding to the client, then select the Set AC Server>Adopt in the context menu. After a few moments its status in the Managed column should go to "YES".
To make a client not being managed by this server:
Right-click on the line corresponding to the client, then select the Set AC Server>Release in the context menu.
To set a custom AC Server for a client:
Right-click on the line corresponding to the client, then select the Set AC Server>Custom in the context menu, then type in the system name (including colon ":" character at the end), on which the other server is running.
To force the synchronization of AC Data on a client:
Make sure that the client is managed by this server (the Managed column indicates "YES". Right click on the client and Use the Force sync host option in the context menu.
To force the synchronization of AC Data to all managed clients:
Use the Force sync all option in the context menu.
Restrictions
  • The panel should only be used on the machine on which the Access Control Server is running.
