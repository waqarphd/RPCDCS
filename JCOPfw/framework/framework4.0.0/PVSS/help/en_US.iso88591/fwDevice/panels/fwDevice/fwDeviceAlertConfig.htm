Panel Name

fwDeviceAlertConfig.pnl
Introduction
 This panel is used to configure the alarms for the elements of the Framework device that is passed as parameter. A device definition is required for the device type/model. The panel allows to make settings for individual elements or for a group of them.
Instructions When the panel is opened a table is displayed with all the elements of the device that can have an alarm. The 'Config' column indicates whether the element has already an alarm defined (Y) or not (N).  The 'Type' column  indicates the data type of the element, and the 'Ranges' column how many ranges are currently defined for the alarm (this is only valid for analog alarms, NA will be displayed when it is not possible to have ranges).

There are three possibilities to change the settings:
  • Individual element
    • By clicking on the name of the element in the table, a pop-up panel will allow you to set the alarm for the element.
  • Selected elements
    • First select the elements on the table. To do this you can use the key combination 'CTRL + click' to select several elements or 'CTRL + SHIFT + click' to select a range in the table. The selected elements have to be of the same type.
    • Click on the 'Configure selected' button. The same panel of the 'Individual element' section will appear, but this time the settings will apply to the selected elements.
  • All elements
    • Click on the 'Configure all' button. The same panel of the 'Individual element' section will appear, but this time the settings will apply to the all the elements in the table. This option will only work if all elements are of the same type.
NOTE: The message below the title indicates whether the device has a summary alarm defined or not. Typically the panel is open in a tab close to another tab with the summary alarm panel, that's why the message tells you to go there to defined the summary. However, it could be that the panel is open independently.
