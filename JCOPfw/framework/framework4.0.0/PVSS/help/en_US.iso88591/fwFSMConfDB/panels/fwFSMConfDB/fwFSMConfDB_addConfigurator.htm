Panel Name

fwFSMConfDB/fwFSMConfDB_addConfigurator.pnl
Introduction

This panel allows the user to add/change all configurator parameters used by the FSMConfDB.

Instructions
APPLY RECIPE
Data cached in PVSS can be applied directly by the devices or by the configurators using the APPLY_RECIPE command. Selecting:
- device: each device unit applies the recipe to itself. → Maximizes flexibility
- configurator: the configurator applies the recipes to all active devices at once. → Optimizes speed
 
The configurator has also a simplified mode, where it applies the same recipe to all enable devices in its domain. This option can only be used in the case where all devices request the same recipe and the recipe name contains the command given by the control unit.
 
If configurator is selected, a timeout should be set. This timeout defines the maximum time in seconds that the configurator waits for the request of all devices to apply the recipe.

By default the configurator option is selected with a Apply recipe timeout of 20 seconds.

LOAD recipes from
FSMConfDB interface can work connected to the ConfDB (need of an Oracle account) or offline
just using caches. Selecting:
- Cache: configurator will load the recipes stored in recipe caches.
- Database: configurator will load the recipes from your Oracle Configuration database.
 
 
Selecting apply recursively sets the previous selection (cache/database) to all configurators children of the given domain.
 
To create/Change a configurator:
Select the configurator options and press OK to complete the action or Cancel to close the panel without creating/modifying the configurator.
