Panel Name

fwAlarmHandlingGroupsConfiguration.pnl
Introduction

This screen is used to configure groups alarms on fwAlarmHandlingGroupsScreen.pnl.
Instructions

Adding a new Group

Right-click on the node "Groups" and selet "Add Group". Give the node a name and optionally a description. Description will appear on the Alarm Screen as tool tip.

A Group can contain the following criterias:

  • system name pattern
  • dp pattern
  • alias pattern
  • FSM Control Unit name

Now that a new group has been created, you can add one or more system names patterns. Right-click on the just created group and select "Add > System Name". Write down a pattern. It can include the asterisk * or it can be a unique, full system name. From this moment the group will show all the alarms belonging to a system whose name matches the pattern you wrote.

It is optionally possible to make a filtering based on dp pattern: Right-click on the just created system, select "Add > Dpe". On this dialog window you can insert as many dpe patterns as you need. Patterns can include the asterisk * as wildchar. A dp selector can help you in writing the dp. The group will then show all the alarms belonging to the system matching the system pattern AND having a dp name matching one of the dpe patterns on the list.

It is also optionally possible to make a filtering based on alias pattern: Right-click on the just created system, select "Add > Alias". On this dialog window you can insert as many alias patterns as you need. Patterns can include the asterisk * as wildchar. The group will then show all the alarms belonging to the system matching the system pattern AND having an alias name matching one of the dpe patterns on the list.

If dp and alias patterns have been introduced under a system, then the group will show thos alarms following the criteria: match: SYSTEM AND (DP OR ALIAS)

It is also possible to make a filtering based on FSM Control Unit names. Right-click on the just created group and select "Add > FSM Control Unit". You can select only the CU belonging to your local system. The Group will then show all the alarms of dps corresponding to the devices under the CU node. This is done in recursive mode: If under the selected CU there is another CU, its devices will also be included in the criteria. If under the selected CU there is another CU imported from another system (i.e. beginning with &), its devices dps, sitting on the remote system, will also be included in the criteria.

Ordering Groups

Each alarm will be included only under the first Group having a matching criteria. You can re-order the Groups by right-clicking the node "Groups" and seleting "Reorder Groups". You can decide to show the groups ordered with your custom order or sorted alphabetically on the Alert Screen. Keep in mind the Groups feeding rule: first-come-first-served. If your criterias ar made such that you are sure that each alarm can only fit to one group, therefore the groups can be sorted in any arbitrary way.

Sound Setup

When a new alarm comes, a sound can be played. To configure it, click on "Panel Setup..." and edit the box "Sound for New Alert". You can Enable it. If you have a sound card and loudspeakers, you can select "Sound file". Else, you can use the "PC speaker". You can avoid the sound to be played too often (i.e. when alarms come and go every few seconds) by introducing an inhibit timer: on "Don't replay sound during" you can write down how many seconds must elapse between to consecutive bells.

Panel size Setup

When resizing the panel at runtime, the widgets are zoomed. This can be useful for example in a big screen. Panel size can be anyway configured without the zooming to have effect. To configure it, click on "Sound/Panel setup..." and edit the box "Panel Size". You can get the width and height of your screen size by pushing the button "Fit to screen size"

Import/Export Settings

The Alarm Screen Groups, Sound and Panel settings can be exported and imported. Just click on the button "Import/Export".

The imported groups will be added to the groups list without deleting the old ones.

On export, the current settings will be first automatically saved.
